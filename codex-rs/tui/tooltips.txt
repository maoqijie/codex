对话变长时使用 /compact 来总结历史并释放上下文。
用 /new 开启一个新想法；之前的会话会保留在历史中。
遇到异常时使用 /feedback 将日志发送给维护者。
用 /model 快速切换模型或推理强度。
用 /permissions 控制 Codex 何时请求确认。
运行 /review 获取当前改动的代码审查。
用 /skills 列出可用技能，或让 Codex 使用某个技能。
用 /status 查看当前模型、审批状态与 token 使用情况。
用 /fork 将当前对话分叉为新线程。
用 /init 生成包含项目专属指引的 AGENTS.md。
用 /mcp 列出已配置的 MCP 工具。
运行 `codex app` 打开 Codex Desktop（在 macOS 上会在需要时自动安装）。
用 /personality 自定义 Codex 的沟通风格。
用 /rename 重命名线程，方便之后恢复。
有 API 问题可使用 OpenAI 文档 MCP；用 `codex mcp add openaiDeveloperDocs --url https://developers.openai.com/mcp` 启用。
加入 OpenAI 社区 Discord：http://discord.gg/openai
访问 Codex 社区论坛：https://community.openai.com/c/codex/37
你可以使用 `!` 在 Codex 中运行任意 shell 命令（例如 `!ls`）。
输入 / 打开命令弹窗；Tab 自动补全斜杠命令。
输入框为空时，按 Esc 返回并编辑上一条消息；Enter 确认。
按 Tab 将消息加入队列而不是立即发送；Enter 总是立即发送。
使用 Ctrl+V 粘贴图片，将其附加到下一条消息。
运行 `codex resume` 可以恢复之前的对话。
